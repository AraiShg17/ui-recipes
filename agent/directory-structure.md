# ディレクトリ設計方針

このプロジェクトでは、**機能単位でまとまりを保つ構成**を採用する。

## 基本ルール

- コンポーネント関連ファイルは、可能な限り同じディレクトリに置く
- 共有ロジックは `lib/` に集約する
- グローバルに効く設定は `src/app/` のレイヤーで管理する
- プロジェクト固有のクラウド/外部サービス前提をディレクトリ規約に埋め込まない

## 推奨トップレベル

```text
src/
  app/            # App Router
  components/     # UI コンポーネント
  lib/            # 共通ロジック・ユーティリティ
public/           # 静的アセット
agent/            # エージェント向けルール
```

## コンポーネント構成

```text
src/components/ComponentName/
  ComponentName.tsx
  ComponentName.module.css
  types.ts            # 必要な場合のみ
  hooks/              # 必要な場合のみ
  api/                # 必要な場合のみ
```

## 配置の判断基準

1. 特定コンポーネント専用のコードか
- Yes: そのコンポーネント配下に置く
- No: 次へ

2. 複数箇所で再利用するか
- Yes: `src/lib/` へ置く
- No: 呼び出し元に近い場所へ置く

## ドキュメント方針

- 全体ルールは `AGENTS.md` と `agent/*.md` に置く
- コンポーネント固有の補足は対象ディレクトリに `README.md` を置いてよい

## 避けること

- コンポーネント専用の Hook/API を無条件にトップレベルへ分離する
- ディレクトリ規約に特定サービス名や環境名を固定で書く
